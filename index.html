从 stow + git 手工符号链接，迁移到 Nix + home-manager 声明式全环境管理。覆盖 Shell、编辑器、Git、Tmux、文件管理器等全部日常工具，CI 自动构建验证，变更可回滚。

包管理

33 个包通过 home.packages 声明，来自 nixpkgs-unstable channel。

  • Shell 工具：bat, fd, fzf, ripgrep, lf, tmux, zsh 等 11 个
  • 编辑器：neovim, nano
  • 版本控制：git, git-crypt, stow
  • 构建工具：gcc, gnumake, tree-sitter
  • 网络实用：podman, curl, ffmpeg, openssh 等 8 个
  • Python：python3, weasyprint（Kami 排版）

程序模块

8 个 home-manager 原生模块，配置声明即生效。

  • starship：极简 prompt，目录 + git 分支
  • bash / zsh：initExtra 外置脚本，fzf 快捷键
  • git：GPG 签名 + gh 凭证助手
  • tmux：vi 模式、256 色、鼠标支持
  • fzf：Ctrl+R 历史 / Ctrl+T 文件 / Alt+C 目录
  • lf：三栏布局 + pv.sh 预览脚本

配置模块一览

模块方式说明
bashprograms.bashinitExtra.bash 外置；history 5000 条
zshprograms.zshinitExtra.zsh 外置；SHARE_HISTORY
gitprograms.gitGPG 签名 + gh 凭证；默认分支 main
tmuxprograms.tmuxvi 键位、256 色、base-index 1
starshipprograms.starship目录 + git 分支 + 状态 + ❯ 提示符
fzfprograms.fzf40% 高度、reverse 布局、文件预览
lfprograms.lf三栏 1:2:3、vim 键位、pv.sh 预览
gnupghome.fileloopback pinentry，配合 GPG 签名
neovimhome.file递归复制整个 nvim 配置目录

迁移路径

阶段 1
stow 管理
GNU stow + git 手工维护符号链接，git-crypt 加密敏感文件
阶段 2
引入 Nix
Nix 包管理器接管系统包安装，统一版本管理
阶段 3
home-manager
声明式全环境：包 + 配置 + 服务，一次 switch 到位，CI 自动验证

日常命令

命令用途
home-manager switch应用配置变更
home-manager build仅构建，测试语法
home-manager generations查看历史版本
nix-shell -p <pkg>临时试用包，无需安装
nix-collect-garbage -d清理旧版本，回收空间
home-manager expire-generations 7d只保留最近 7 天的版本
注意事项：shellOptions 只接受 shopt 参数，set -o 项（如 noclobber）放 initExtra；Nix 字符串不支持 ${}，含 bash 变量的内容用 builtins.readFile 外置文件；Channel 锁定 nixpkgs-unstable + home-manager master；CI 通过 GitHub Actions 自动 home-manager build + nixpkgs-fmt 检查。
