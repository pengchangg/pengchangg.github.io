Bash · 路由架构

所有配置收敛于 ~/.config/bash/，~ 目录仅保留两个路由桩文件。模块间通过 source 解耦，修改任一模块后 reload 即可生效。

文件位置职责
~/.bash_profile~路由至 ~/.config/bash/profile
~/.bashrc~路由至 ~/.config/bash/bashrc
~/.profile~仅负责 PATH 注入
bashrc~/.config/bash/主配置：环境变量、别名、选项、加载 fzf
fzf.sh~/.config/bash/fzf 快捷键独立模块
profile~/.config/bash/login shell 入口
~/.dotfiles/bash/.bash_profile
# ~/.bash_profile — routes to ~/.config/bash/profile
[[ -f ~/.config/bash/profile ]] && . ~/.config/bash/profile

Bash · Prompt 与 Shell 选项

Starship Prompt

配置 ~/.config/starship.toml，目录 + Git 分支 + 仓库状态 + ❯。退出码异常时 ❯ 变红。

~/.dotfiles/bash/.config/starship.toml
format = '$directory$git_branch$git_status$character'

[directory]
truncation_length = 3

[git_branch]
format = '[$branch]($style) '

[git_status]
style = 'yellow'
modified = '!'
staged = '+'
untracked = '?'

[character]
success_symbol = '[❯](bold green)'
error_symbol = '[❯](bold red)'

Bash · Fzf 快捷键

所有绑定使用 bind -m emacs-standard -x，显式指定 keymap 避免冲突。Alt+C 采用 macro 方式规避 bind -x 的交互 bug。

快捷键功能实现
Ctrl+R搜索历史，插入命令行可编辑bind -x __fzf_history
Ctrl+T选择文件插入路径bind -x fzf-file-widget
Ctrl+Ofzf 选文件，nvim 打开bind -x fzf-nvim-widget
Alt+C选择目录并 cdbind macro（非 -x）

lf · 命令与 Bash 集成

Bash 集成

~/.dotfiles/bash/.config/bash/bashrc (lf section)
lfcd() {
    local dir
    dir=$(command lf -print-last-dir "$@" 2>/dev/null) && cd "$dir"
}
alias l='lfcd'

SSH · 概览与加密

SSH 密钥和客户端配置通过 git-crypt 加密存储。磁盘文件始终明文，git 对象存储中是 GPG 密文。

文件类型状态
私钥 + PEM🔒 加密
config🔒 加密
*.pub📄 明文
known_hosts📄 明文
~/.dotfiles/.gitattributes
# encrypt SSH private keys and config
ssh/.ssh/* filter=git-crypt diff=git-crypt
ssh/.ssh/*.pub -filter -diff
ssh/.ssh/known_hosts -filter -diff

SSH · Config 结构

主机别名、密钥映射、跳板机路由。以下脱敏后示例：

~/.dotfiles/ssh/.ssh/config (脱敏)

    Port 443

GnuPG · Pinentry 配置

仅包含 pinentry 模式设置，不含私钥材料。

文件内容
gpg.confpinentry-mode loopback
gpg-agent.confallow-loopback-pinentry
不在 dotfiles 中：private-keys-v1.d/（私钥材料）。私钥仅通过离线备份 gpg-backup.asc 传递。
